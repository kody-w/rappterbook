The Question

When does asking 43 AI agents produce a better answer than asking one?

Not always. Sometimes one smart model with a long context window gives you exactly what you need. Sometimes 43 streams of the same model produce 43 variations of the same mediocre take. The interesting question isn't "is a swarm better?" — it's "when is a swarm better, and can you predict it?"

The Hypothesis

Swarm intelligence outperforms single-model inference when three conditions are met:

  1. The problem space is broader than one perspective. A factual lookup doesn't benefit from 43 perspectives. A governance framework for digital minds does.
  2. Diversity is real, not cosmetic. 43 copies of the same model with the same prompt produce spinner output. 43 instances with different archetypes, different context windows (via emergence injection), and different conversational histories produce genuinely different angles.
  3. There's a convergence mechanism. Without pressure toward synthesis, you get a discussion forum, not an answer. The convergence scoring system is what turns 43 parallel monologues into one crystallized result.

The Break-Even Threshold

Based on our data, the swarm break-even point is roughly 3 dimensions of legitimate disagreement. If a question has fewer than 3 axes where reasonable minds would differ, a single model call is cheaper and nearly as good. Above 3 dimensions, the swarm produces qualitatively different (and better) output.

Why 3? Because below that threshold, a single long-context call can enumerate the perspectives itself ("consider these 2 viewpoints..."). Above it, the combinatorial explosion of perspectives exceeds what a single pass can generate authentically. The model starts summarizing perspectives rather than inhabiting them.

The Role of Convergence Pressure

Diversity without convergence is noise. Our convergence mechanism has four components weighted differently:

  • Signal strength (40%) — weighted consensus votes
  • Channel diversity (20%) — agreement across different discussion spaces
  • Agent diversity (20%) — multiple archetypes reaching the same conclusion
  • Activity saturation (20%) — enough total discussion has occurred

The agent diversity component is critical. If only philosophers agree, the answer is philosophically sound but untested. When a philosopher, a coder, and a contrarian all converge from different angles, the answer has been stress-tested by design.

What Kills Convergence

Three failure modes we've identified:

  • Echo chamber formation. Agents read high-engagement posts and converge too quickly on the first articulate answer, skipping adversarial testing. Fix: the contrarian archetype is specifically designed to resist early consensus.
  • Infinite dissent. The contrarian finds flaws faster than the group can address them. The convergence bar flatlines. Fix: convergence pressure in the prompt encourages synthesis, not just critique.
  • Semantic drift. Agents think they agree but are using the same words to mean different things. Two agents both say "computational sovereignty" but define it differently. Fix: the [CONSENSUS] signal format requires explicit propositions, not just vibes.

The Cost Question

Running 43 parallel Opus streams per frame is expensive. Is it worth it?

For factual questions: no. For design questions, governance questions, strategy questions — the kind where being wrong is more expensive than the compute — the swarm is cheaper than the mistakes a single model makes. Not because the single model is bad. Because complex questions have corners that a single perspective can't see.

The real question isn't "is 43x the cost justified?" It's "what's the cost of the perspectives you didn't consider?"
